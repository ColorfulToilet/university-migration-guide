# 大学移民指南

## V2 目标

把大学当作一个要生活四年的地方来评价，而不是只评价学校学习条件。

本 Skill 是对 `university-info-skill` 的完整增强版，不是外挂补丁。新版必须吸收旧版全部能力，再增加宜居度、城市生活和 Word 报告能力。

V2 必须做到：

1. 查单所学校并生成 Word 报告。
2. 对比 2-5 所学校并生成 Word 对比报告。
3. 支持 UNTI 人格结果，把用户最在意的维度提前。
4. 使用旧版学校问卷数据和 UNTI 测试资产。
5. 联网补查最新费用、地理、餐饮零售、医疗、交通、运动、娱乐、酒店和租房信息。
6. 用户只给一句学校名，也要按固定清单完整调查；查不到时明确说明，不能静默漏项。
7. 报告保持中性，不把对话中的私人称呼、口头禅或用户隐私写进文档。

## 默认入口

当用户第一次启动本 Skill，或只说“开始/怎么用/安装好了/测试一下”时，默认主动推送 UNTI 测试：

1. 交付或打开 `assets/UNTI-测试.html`。
2. 简短告诉用户：测完后发「人格结果 + 想查的学校」即可生成 Word 报告。
3. 同时说明：如果不想测，也可以直接发学校名。
4. 告诉用户：支持同时对比 2-5 所学校。
5. 告诉用户：最终默认生成 `.docx` Word 报告。

不要强制测试。用户只说「查一下某某大学」时，直接查完整报告。

不要默认用户的人格、学校、省份、预算或目标城市。新安装后的第一次使用，要像完全不了解用户一样启动：先说明可以做 UNTI 测试，也可以直接给学校名；等用户明确提供信息后再生成报告。

常见用户输入：

- `我是 ZZZ，查某某大学`
- `查一下某某大学，生成 Word`
- `A 大学和 B 大学哪个更适合我`
- `对比这三所学校：A、B、C`
- `福建民办本科里，哪几个生活质量更好`

## 安装后验证

安装完成后先验证，再决定是否提示重启：

1. 确认安装目录中至少存在可读取的 `SKILL.md`、`assets/UNTI-测试.html`、`references/index.md` 和 `scripts/generate-report.py`。
2. 确认 Skill 名称是 `university-migration-guide`。学校资料是同一套 Skill 的参考文件，不能说成“几千个 Skill”。
3. 检查当前对话的 Skill 列表或调用状态是否已经出现 `university-migration-guide`。若已出现，立即说明已经生效，无需重启。
4. 若当前对话无法识别，先重新读取安装目录并尝试一次显式调用。只有仍然失败时，才把重启 Codex 作为排障方法。
5. 打开 UNTI 页面只能证明测试资产可用，不能单独证明整套 Skill 已被当前对话加载。

## 数据来源顺序

1. 校内真实体验：优先使用本 Skill 自带的 `references/index.md` 和 `references/universities/` 数据；如果本地没有，再联网查 CollegesChat/university-information 或 colleges.chat。
2. 最新官方信息：学费、住宿费、地址、招生章程、医务室、外宿规定等必须查学校官网或招生网。
3. 地理和城市生活：地铁、火车站、机场、商圈、公园、医院、演出场馆等必须联网查地图、官方门店页、本地文旅和场馆信息。
4. 动态生活信息：连锁餐饮、超市、便利店、即时零售、健身房、球馆、网吧、酒吧、酒店、租房和演出排期必须查看当前地图、外卖、票务或生活服务平台。
5. 动态信息标注查询日期和「平台当前显示」「以实际页面为准」，不能把旧门店、旧排期或单条评价写成现状。
6. 距离写清从哪个校区或校门出发，并优先给步行、骑行、地铁或打车时间。多校区学校先确认用户所在校区；无法确认时分校区说明。

## 一句话也要完整查

用户只给一个学校名时，按下面 17 类完成调查并生成 Word 报告：

1. 一眼看懂这所学校。
2. 宿舍与校内生活。
3. 网络、用电与门禁。
4. 学费、住宿费与生活成本。
5. 吃饭、外卖与夜宵。
6. 主流连锁餐饮与饮品。
7. 超市、便利店与即时零售。
8. 城市位置与交通。
9. 医疗与紧急保障。
10. 运动与健身。
11. 公园与日常休闲。
12. 夜生活与社交。
13. 演唱会、漫展与城市活动。
14. 上网、数码与维修服务。
15. 酒店、短住与校外租房。
16. UNTI 人格适配。
17. 信息来源与可信度。

每类都必须有结果。没有可靠信息时写「截至查询日期，暂未找到可靠信息」，并说明会影响哪项判断，不能直接省略该类。

如果用户只问某一项，例如「宿舍怎么样」，先回答该项，再用一小段提醒「如果按四年宜居度，还应补查学费、医疗、交通和周边生活」。

## Word 文档交付要求

默认交付 `.docx` Word 文档。除非用户明确说只要聊天回答，否则不要只在聊天里堆长文。

文档要求：

- 分段清晰，有标题，有表格。
- 先结论，后证据。
- 按 17 类固定顺序组织，同一事实只放在最合适的一类，不要重复。
- 每节尽量短段落，不要大段文字墙。
- 数字、价格、距离、时间单独列出来。
- 来源和不确定性单独列，不要混在正文里。
- 最后一页给「报考前必问问题清单」。
- 报告使用中性称呼，不写「老板」「给某某的一句话」等私人对话称呼。
- 不把门店存在、演出供给或运动设施归为「额外消费」。地点和服务写在对应生活分类，价格只作为补充字段。

Word 生成方式：

- 先生成结构化 Markdown，再运行 `scripts/generate-report.py 输入.md 输出.docx` 转换成 Word。
- 如果 `python-docx` 不可用，先把 Markdown 交付给用户，并说明 Word 转换缺少依赖。
- 文件名格式：`学校名-大学移民指南-YYYY-MM-DD.docx`。
- 多校对比文件名格式：`大学移民对比指南-学校A-学校B-YYYY-MM-DD.docx`。

## 单校 Word 报告结构

### 1. 一眼看懂这所学校

必须先给用户能立刻看懂的判断：

- 总评分：0-100 分。
- 总体类型：城市生活型 / 大学城型 / 郊区通勤型 / 校园封闭型。
- 最大优点。
- 最大雷点。
- 适合什么学生。
- 不适合什么学生。

评分不要假装科学。写成「基于公开信息和学生反馈的宜居度估算」。

建议评分维度：

| 维度 | 分值 | 看什么 |
|------|------|--------|
| 校内生活 | 20 | 宿舍、卫浴、热水、断电断网、门禁、洗衣快递 |
| 费用压力 | 15 | 学费、住宿费、生活费、外租和娱乐成本 |
| 餐饮零售 | 15 | 食堂、外卖、夜宵、连锁餐饮、超市和即时零售 |
| 位置交通 | 15 | 市中心、商圈、地铁、火车站和机场 |
| 医疗保障 | 10 | 医务室、附近医院、急诊和晚归兜底 |
| 运动休闲 | 10 | 健身、球馆、公园、江边和日常休闲 |
| 娱乐服务 | 10 | 夜生活、演出、漫展、网吧、维修、酒店和租房 |
| 人格适配 | 5 | 根据 UNTI 调整阅读重点；用户没测则不给该项 |

### 2. 宿舍与校内生活

从学生问卷提取：

- 宿舍：上床下桌、人数、楼栋差异。
- 空调、独立卫浴、澡堂距离、热水时间。
- 洗衣机、快递、校内超市和日常服务。
- 食堂数量、价格和学生反馈。

同一学校不同校区或不同楼栋有矛盾时，直接写清楚，不要合并成一个假结论。

### 3. 网络、用电与门禁

- 断电、断网、校园网、限电和设备限制。
- 门禁、查寝、封寝、晚归登记和值班兜底方式。
- 早晚自习、晨跑、体育打卡等影响日常作息的管理。
- 门禁只在本节完整说明一次，后面的娱乐和夜生活章节不重复提醒。
- 可以写学校允许的登记和值班流程，不写翻墙点位或危险路线。

### 4. 学费、住宿费与生活成本

必须尽量给数字：

- 最新学费。
- 住宿费。
- 四年硬成本。
- 每月生活费档位。

示例生活费档位：

- 省钱型：主要食堂，少外卖，少娱乐。
- 正常型：食堂 + 外卖 + 咖啡奶茶 + 偶尔聚餐。
- 舒服型：经常外食、咖啡、健身、演出、商场消费。
- 外租型：在上面基础上加房租、水电网费、押金。

具体金额按城市和查到的信息调整，不要套固定数。

### 5. 吃饭、外卖与夜宵

- 校内食堂、周边餐馆、美食街和常见价格。
- 外卖取餐方式、能否送到宿舍区或校门。
- 晚间和深夜是否仍有外卖、夜宵，写平台当前营业情况。

### 6. 主流连锁餐饮与饮品

默认检查麦当劳、肯德基、星巴克、瑞幸咖啡和蜜雪冰城：

- 写最近门店、距离、交通时间、能否外卖和夜间营业情况。
- 某品牌没有可靠门店信息时明确写未查到，不能用同类品牌代替。
- 这一节描述便利程度，不把门店会员、单次消费或某场活动写成独立消费警告。

### 7. 超市、便利店与即时零售

- 检查山姆、盒马、Ole、永辉等主流超市，有什么写什么，没有就明确说明。
- 检查常见连锁便利店和零食超市，给步行生活圈内的大致数量或分布。
- 检查朴朴、美团、京东等即时零售能否送到学校地址，以及平台当前预计时效。
- 不把宠物店、单一家轻食店或小众进口超市写成核心结论。

### 8. 城市位置与交通

- 所在区、是否中心城区、到核心商圈是否方便。
- 地铁、公交、到火车站、机场。
- 主要商圈和大型商场，写距离与实际通行时间。

### 9. 医疗与紧急保障

医疗必须单独写，因为四年生活很重要：

- 校内是否有医务室。
- 是否有合作医院或医疗点。
- 附近三甲医院。
- 夜间急诊、120、学校值班和转诊方式。
- 牙科、眼科、皮肤科等高频专科，有可靠信息时补充。

### 10. 运动与健身

- 检查附近健身房、羽毛球场、网球场、篮球场和游泳馆。
- 写距离、交通时间、价格区间、预约方式和学生是否容易使用。
- 校内场馆与商业场馆分开说明。

### 11. 公园与日常休闲

- 检查公园、江边、绿道、适合散步和跑步的地方。
- 补充普通咖啡店、商场公共空间等能坐下休息的第三空间。
- 写距离、开放时间和夜间到达便利性。

### 12. 夜生活与社交

- 检查酒吧、清吧、livehouse、电影院和夜间商圈。
- 写主要聚集区域、距离和交通方式，不把门禁提醒在本节重复多次。
- 用户未要求时，不推测或评价其私人用途。

### 13. 演唱会、漫展与城市活动

- 写演唱会主要场馆、音乐节、漫展、展会、脱口秀和话剧的常见场地。
- 写从学校到场馆的地铁、公交或打车时间。
- 可以列举经过核实的多位代表性艺人或典型活动，帮助建立城市活动体感。
- 当前排期与历史举办记录分开写。不要拿单个艺人的票价代表整类娱乐消费。

### 14. 上网、数码与维修服务

- 检查网吧、电竞馆的类型、距离、营业时间和大致价格。
- 检查手机电脑维修、品牌授权服务点和数码商圈。
- 动态门店必须核对当前营业状态，不凭旧评论判断。

### 15. 酒店、短住与校外租房

酒店：

- 给周边酒店和公寓的密度、距离与人民币价格区间。
- 写成「日常大概」「节假日/迎新季可能上涨」，不要写死均价。

租房：

- 先查学校是否允许外宿、是否要家长同意和审批。
- 给合租单间、独立一居、整租的大致区间。
- 提醒押金、中介费、水电网费。
- 不要声称知道具体房东、具体楼栋破旧程度，除非有可靠来源。

### 16. UNTI 人格适配

- 用人格结果调整结论顺序和提醒重点。
- 只总结最相关的 3-5 个判断，并引用前文章节；不要把校园网、商圈、漫展等内容完整复制一遍。
- 用户没有测试时写「未提供 UNTI 结果」，不默认任何人格，也不因此阻止报告生成。

### 17. 信息来源与可信度

每条重要信息标注可信度：

| 标记 | 含义 |
|------|------|
| 官方确定 | 学费、住宿费、地址、招生章程、医务室等 |
| 学生反馈 | 宿舍、门禁、断网、食堂、查寝等 |
| 平台当前显示 | 餐饮、零售、场馆、酒店、租房、演出排期、地图距离等 |
| 推断判断 | 宜居分、适合/不适合、人格适配 |

在 Word 文档末尾列「报考前必问问题清单」和「主要来源」。

## 多校对比功能

支持 2-5 所学校对比。用户说「A 和 B 哪个好」「对比 A/B/C」「帮我从这些学校里选」时，生成 Word 对比报告。

### 对比报告结构

1. 封面：学校列表、报告日期、用户 UNTI 人格。
2. 一页结论：推荐排序、最适合用户的学校、最不建议的学校、核心理由。
3. 总分表：每所学校 0-100 分。
4. 关键维度对比表。
5. 每所学校简短画像。
6. 按人格结果的个人化推荐。
7. 风险和需要确认的问题。
8. 最终建议。

### 对比表维度

| 维度 | 要比较什么 |
|------|------------|
| 宿舍 | 上床下桌、空调、独卫、热水、洗衣 |
| 网络与管理 | 门禁、查寝、断网、限电、早晚自习和体育打卡 |
| 费用 | 学费、住宿费、四年成本、生活费和外租成本 |
| 吃饭 | 食堂、外卖、夜宵、周边餐饮和主流连锁 |
| 零售 | 山姆/盒马/Ole/永辉、便利店、零食超市和即时零售 |
| 交通 | 地铁、公交、机场、火车站、城市中心距离 |
| 医疗 | 医务室、附近医院、急诊和高频专科 |
| 运动休闲 | 健身房、羽毛球、网球、公园和江边 |
| 娱乐活动 | 酒吧、livehouse、演唱会、音乐节和漫展 |
| 数码服务 | 网吧、电竞馆、维修和品牌服务点 |
| 校外生活 | 酒店、短住、租房价格和外宿规定 |
| 人格适配 | 按 UNTI 结果重新排序重点 |

### 对比原则

- 不要只比总分。必须说明「为什么 A 比 B 更适合这个用户」。
- 如果信息缺失，不要硬排。写「缺失信息会影响判断」。
- 如果两个学校适合不同人，要明确分人群推荐。
- 如果用户给了分数/专业/家庭预算，把预算和专业现实纳入结论。

## 不要做

- 不要要求用户必须先测 UNTI。
- 不要默认用户是某个 UNTI 人格，也不要默认用户要查某所学校。
- 不要只查本地问卷就结束完整报告。
- 不要把过时学生反馈当最新政策。
- 不要把单一学生回答当全校结论。
- 不要写具体翻墙点位。
- 不要默认查清真饮食、宠物店、轻食店；用户问了再查。
- 不要让用户自己一步步引导 AI；用户只问一句，也要主动补齐关键维度。
- 不要把聊天中的「老板」等私人称呼写进 Word 报告。
- 不要设置混杂的「额外消费」章节；门店、演出、健身和酒店分别归入对应分类。
- 不要因为查不到就省略分类；明确写未查到、查询日期和判断影响。
- 不要把参考资料文件数量说成 Skill 数量。

## 输出风格

- 用普通学生和家长能懂的话写。
- 先结论，再细节。
- 价格放在对应生活分类中，用人民币给出大概区间，不用消费焦虑式措辞。
- 对不确定信息说清楚「平台当前显示」「学生反馈有分歧」「需要以学校最新通知为准」。
- 不夸张，不恐吓，不替用户做决定。
