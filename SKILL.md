# 大学四年宜居度查询

## V1 目标

把大学当作一个要生活四年的地方来评价，而不是只评价学校学习条件。

本 Skill 是对 `university-info-skill` 的完整增强版，不是外挂补丁。新版必须吸收旧版全部能力，再增加宜居度、城市生活和 Word 报告能力。

V1 先做到：

1. 查单所学校并生成 Word 报告。
2. 对比 2-5 所学校并生成 Word 对比报告。
3. 支持 UNTI 人格结果，把用户最在意的维度提前。
4. 使用旧版学校问卷数据和 UNTI 测试资产。
5. 联网补查最新官方费用、地理、医疗、交通、商圈、演出、酒店、租房等信息。

以后再慢慢优化评分权重、模板美观、自动地图和更细的数据表。

## 默认入口

当用户第一次来问大学生活，可以推荐：

1. 先做 UNTI 大学生活人格测试。
2. 测完后发「人格结果 + 想查的学校」。

但不要强制。用户只说「查一下某某大学」时，直接查完整报告。

常见用户输入：

- `我是 ZZZ，查福州大学至诚学院`
- `查一下厦门大学嘉庚学院，生成 Word`
- `福州大学至诚学院和厦门大学嘉庚学院哪个更适合我`
- `对比这三所学校：A、B、C`
- `福建民办本科里，哪几个生活质量更好`

## 数据来源顺序

1. 校内真实体验：优先使用本 Skill 自带的 `references/index.md` 和 `references/universities/` 数据；如果本地没有，再联网查 CollegesChat/university-information 或 colleges.chat。
2. 最新官方信息：学费、住宿费、地址、招生章程、医务室、外宿规定等必须查学校官网或招生网。
3. 地理和城市生活：地铁、火车站、机场、商圈、公园、酒店、租房、维修、医院、演出场馆等必须联网查地图、官方门店页、票务、本地文旅/场馆信息。
4. 平台信息：酒店、租房、门店、演出排期等会变化，回答时标注「平台当前显示」「以实际页面为准」。

## 一句话也要完整查

用户只给一个学校名时，不要只回答宿舍或食堂。默认生成完整 Word 报告：

1. 5 行结论。
2. 校内生活。
3. 钱。
4. 周边城市生活半径。
5. 医疗、安全和晚归。
6. 娱乐和社交。
7. 租房、酒店和短住。
8. 月生活费档位。
9. 适合谁、不适合谁。

如果用户只问某一项，例如「宿舍怎么样」，先回答该项，再用一小段提醒「如果按四年宜居度，还应补查学费、医疗、交通和周边生活」。

## Word 文档交付要求

默认交付 `.docx` Word 文档。除非用户明确说只要聊天回答，否则不要只在聊天里堆长文。

文档要求：

- 分段清晰，有标题，有表格。
- 先结论，后证据。
- 逻辑顺序固定，不要一会儿讲宿舍、一会儿跳去演唱会、一会儿又回学费。
- 每节尽量短段落，不要大段文字墙。
- 数字、价格、距离、时间单独列出来。
- 来源和不确定性单独列，不要混在正文里。
- 最后一页给「报考前必问问题清单」。

Word 生成方式：

- 先生成结构化 Markdown，再运行 `scripts/generate-report.py 输入.md 输出.docx` 转换成 Word。
- 如果 `python-docx` 不可用，先把 Markdown 交付给用户，并说明 Word 转换缺少依赖。
- 文件名格式：`学校名-大学四年宜居度报告-YYYY-MM-DD.docx`。
- 多校对比文件名格式：`大学宜居度对比报告-学校A-学校B-YYYY-MM-DD.docx`。

## 单校 Word 报告结构

### 1. 开头 5 行结论

必须先给用户能立刻看懂的判断：

- 总评分：0-100 分。
- 总体类型：城市生活型 / 大学城型 / 郊区通勤型 / 校园封闭型。
- 最大优点。
- 最大雷点。
- 适合什么学生。
- 不适合什么学生。

评分不要假装科学。写成「基于公开信息和学生反馈的宜居度估算」。

建议评分维度：

| 维度 | 分值 | 看什么 |
|------|------|--------|
| 校内生活 | 25 | 宿舍、空调、卫浴、热水、断电断网、食堂外卖、洗衣快递 |
| 费用压力 | 15 | 学费、住宿费、生活费、外租和娱乐成本 |
| 城市便利 | 20 | 地铁、商圈、火车站机场、即时零售、维修、酒店 |
| 医疗安全 | 15 | 医务室、附近医院、晚归兜底、安全风险 |
| 娱乐社交 | 15 | 夜宵、咖啡、酒吧、livehouse、演唱会、漫展、公园 |
| 人格适配 | 10 | 根据 UNTI 结果调整；用户没测则不给该项或按中性估 |

### 2. 校内生活

从学生问卷提取：

- 宿舍：上床下桌、人数、楼栋差异。
- 空调、独立卫浴、澡堂距离、热水时间。
- 断电断网、校园网、限电。
- 门禁、查寝、封寝、晚归。
- 食堂、外卖、洗衣机、快递、超市。
- 早晚自习、晨跑、跑步打卡。

同一学校不同校区或不同楼栋有矛盾时，直接写清楚，不要合并成一个假结论。

### 3. 钱

必须尽量给数字：

- 最新学费。
- 住宿费。
- 四年硬成本。
- 山姆会员、演唱会、健身/球馆、酒店、外租等是否额外花钱。
- 每月生活费档位。

示例生活费档位：

- 省钱型：主要食堂，少外卖，少娱乐。
- 正常型：食堂 + 外卖 + 咖啡奶茶 + 偶尔聚餐。
- 舒服型：经常外食、咖啡、健身、演出、商场消费。
- 外租型：在上面基础上加房租、水电网费、押金。

具体金额按城市和查到的信息调整，不要套固定数。

### 4. 周边生活半径

重点查稳定、对多数学生有用的信息：

- 所在区、是否中心城区、到核心商圈是否方便。
- 地铁、公交、到火车站、机场。
- 大型商场、主流优质超市：山姆、Ole、盒马、万象城、永辉等。只写能查到的，不要凭印象。
- 朴朴、京东买药、美团买药等即时零售。
- 餐饮、外卖、夜宵、美食街。
- 咖啡和第三空间：瑞幸、星巴克、Manner、商场咖啡、能坐的公共空间。
- 手机电脑维修、品牌授权服务点、数码商圈。
- 公园、江边、适合散步和跑步的地方。

不要把宠物店、单一家轻食店、小众进口超市写成核心结论。它们易倒闭、受众窄，用户明确问了再查。

### 5. 医疗、安全和晚归

医疗必须单独写，因为四年生活很重要：

- 校内是否有医务室。
- 是否有合作医院或医疗点。
- 附近三甲医院。
- 牙科、眼科、皮肤科等高频专科，有需要时补查。

安全和晚归：

- 门禁只点一次，不要在每个娱乐项里反复提醒。
- 可以写「有登记、宿管/值班流程等兜底方式」，但不要写具体翻墙地点，不要鼓励违规路线。
- 宿舍楼道监控、推销、偷盗等如果学生反馈提到，可以作为风险写入。

### 6. 娱乐和社交

大学生会在意城市生活，不要只看学习：

- 酒吧、清吧、livehouse、夜宵街。
- 演唱会主要场馆，以及学校过去是否在合理打车/地铁半径内。
- 音乐节、漫展、展会、脱口秀、话剧。
- 电影院、商场、江边散步、适合约会和社交的区域。

可以列举城市常见演出场馆和曾办过的典型明星/活动，帮助用户建立感觉。明星名单和排期必须联网核对，不要凭记忆。

### 7. 酒店、短住和租房

酒店：

- 给周边酒店/公寓密度和价格区间。
- 写成「日常大概」「节假日/迎新季可能上涨」，不要写死均价。

租房：

- 先查学校是否允许外宿、是否要家长同意和审批。
- 给合租单间、独立一居、整租的大致区间。
- 提醒押金、中介费、水电网费。
- 不要声称知道具体房东、具体楼栋破旧程度，除非有可靠来源。

### 8. 来源和可信度

每条重要信息标注可信度：

| 标记 | 含义 |
|------|------|
| 官方确定 | 学费、住宿费、地址、招生章程、医务室等 |
| 学生反馈 | 宿舍、门禁、断网、食堂、查寝等 |
| 平台当前显示 | 酒店、租房、门店、演出排期、地图距离等 |
| 推断判断 | 宜居分、适合/不适合、人格适配 |

在 Word 文档末尾列「主要来源」。

## 多校对比功能

支持 2-5 所学校对比。用户说「A 和 B 哪个好」「对比 A/B/C」「帮我从这些学校里选」时，生成 Word 对比报告。

### 对比报告结构

1. 封面：学校列表、报告日期、用户 UNTI 人格。
2. 一页结论：推荐排序、最适合用户的学校、最不建议的学校、核心理由。
3. 总分表：每所学校 0-100 分。
4. 关键维度对比表。
5. 每所学校简短画像。
6. 按人格结果的个人化推荐。
7. 风险和需要确认的问题。
8. 最终建议。

### 对比表维度

| 维度 | 要比较什么 |
|------|------------|
| 钱 | 学费、住宿费、四年成本、生活费、外租成本 |
| 宿舍 | 上床下桌、空调、独卫、热水、洗衣 |
| 管理 | 门禁、查寝、断网、早晚自习、跑步打卡 |
| 吃 | 食堂、外卖、夜宵、周边餐饮 |
| 交通 | 地铁、公交、机场、火车站、城市中心距离 |
| 医疗 | 医务室、附近三甲、牙科眼科 |
| 生活 | 商圈、超市、即时零售、维修、酒店 |
| 娱乐 | 酒吧、livehouse、演唱会、漫展、公园江边 |
| 人格适配 | 按 UNTI 结果重新排序重点 |

### 对比原则

- 不要只比总分。必须说明「为什么 A 比 B 更适合这个用户」。
- 如果信息缺失，不要硬排。写「缺失信息会影响判断」。
- 如果两个学校适合不同人，要明确分人群推荐。
- 如果用户给了分数/专业/家庭预算，把预算和专业现实纳入结论。

## 不要做

- 不要要求用户必须先测 UNTI。
- 不要只查本地问卷就结束完整报告。
- 不要把过时学生反馈当最新政策。
- 不要把单一学生回答当全校结论。
- 不要写具体翻墙点位。
- 不要默认查清真饮食、宠物店、轻食店；用户问了再查。
- 不要让用户自己一步步引导 AI；用户只问一句，也要主动补齐关键维度。

## 输出风格

- 用普通学生和家长能懂的话写。
- 先结论，再细节。
- 明确说哪些需要钱、大概多少钱。
- 对不确定信息说清楚「平台当前显示」「学生反馈有分歧」「需要以学校最新通知为准」。
- 不夸张，不恐吓，不替用户做决定。
